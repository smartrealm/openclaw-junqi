# Gateway 生命周期健壮性审计

## 严重问题

### BUG-GL07 · 重启命令超时后与托管兜底并行

**位置**：`src-tauri/src/commands/gateway.rs`

`openclaw gateway restart` 超时或等待失败后，当前实现直接进入托管子进程兜底。重启 CLI 的 `Child` 要等整个函数返回才会因 `kill_on_drop` 被清理，因此兜底启动期间旧命令仍可能操作 LaunchAgent/systemd。

**影响**：系统服务和托管子进程并行抢占同一端口，产生重复重启、端口抖动和不一致的运行模式。

**修复**：所有异常分支先显式终止并等待重启 CLI，再进入托管兜底。

### BUG-GL08 · 操作锁竞争被错误当作重复重启

**位置**：`src-tauri/src/commands/gateway.rs`

`restart_gateway` 获取不到全局操作锁时，直接假定持锁者也是一次重启；但实际持锁者可能是启动、停止、Docker 切换或存储迁移。等待结束后直接返回状态会吞掉用户请求的重启。

**影响**：UI 报告重启结束，实际没有执行重启；配置变更可能仍未生效。

**修复**：增加可判定的重启完成代次。只有等待期间确实完成过另一次重启才能合并，否则获得操作锁后继续执行本次重启。

### BUG-GL09 · 旧异步结果可覆盖新的连接生命周期

**位置**：`src/services/gateway/GatewayConnectionManager.ts`

状态探测、启动命令和连接目标解析都是异步操作。`reset()`、`reconnect()` 或 `destroy()` 后，旧 Promise 仍可回写 FSM 或重新打开 WebSocket。

**影响**：手动重连被旧探测降级、销毁后的窗口重新连接、旧启动结果推动新状态机，最终表现为长时间“正在连接”。

**修复**：Manager 持有单调递增的生命周期代次；所有异步动作只允许在发起代次仍有效时提交结果。

## 中等问题

### BUG-GL10 · 2 秒状态轮询允许并发和乱序回调

**位置**：`src/api/tauri-adapter.ts`

每次状态轮询包含进程状态、端口探测和日志读取，耗时可能超过 2 秒。`setInterval` 不等待上一次完成，旧轮询可能晚于新轮询回调。

**影响**：健康状态短暂倒退，重复触发 START/CONNECT，日志与 retrying 状态闪烁。

**修复**：使用自调度串行轮询，并用轮询代次丢弃停止订阅前后产生的过期结果；异步注册迟于 cleanup 完成的 Tauri 监听器立即自解除。
