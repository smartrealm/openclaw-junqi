# Dashboard 数据与操作区修复规格

## BUG-DASH-01 · 统一智能体名称

**目标**：活跃智能体、活动列表和会话界面使用同一展示名称优先级。

**验收**：

- [x] 配置了 `name` 时优先显示 `name`。
- [x] 只有 `identity.name` 时显示身份名称。
- [x] 两者都没有时才使用内置翻译或 ID。
- [x] 展示名称不参与会话路由 key。

## BUG-DASH-02 · 全智能体统计口径

**目标**：Dashboard 的费用和用量统计覆盖所有智能体，与可见会话范围一致。

**验收**：

- [x] `usage.cost` 请求显式携带 `agentScope: "all"`。
- [x] `sessions.usage` 请求显式携带 `agentScope: "all"`。
- [x] 全量分析费用请求使用相同范围。
- [x] 费用类型覆盖 OpenClaw 的缓存费用、总 Token 和缺失价格计数。

## BUG-DASH-03 · 保留零费用日期轴

**目标**：只要 Gateway 返回合法日期桶，费用图就显示 X 轴。

**验收**：

- [x] 全零费用的日期桶仍生成图表点。
- [x] 日期按升序排序并只保留最近 14 天。
- [x] 空数组才进入“暂无费用数据”状态。
- [x] 非数字和负数不会污染图表。

## BUG-DASH-04 · 活动信息完整可扫读

**目标**：每条会话活动同时呈现会话、智能体、模型、Token 和时间。

**验收**：

- [x] 条目按统一活跃时间降序排列。
- [x] 模型名去掉 provider 前缀但保留完整 tooltip。
- [x] 显示本地时间，完整日期时间可悬停查看。
- [x] 点击会话活动仍打开对应会话。
- [x] 没有时间的累计压缩数不再伪装成最新活动。

## BUG-DASH-05 · 扩展快速操作

**目标**：控制中心提供六个真实且稳定的操作入口。

**验收**：

- [x] 支持新建主智能体会话。
- [x] 支持打开智能体、用量分析、技能和系统状态页面。
- [x] 保留当前会话压缩及成功/失败反馈。
- [x] 路由入口遵循 edition 功能开关。
- [ ] 窄屏和桌面布局均无文字溢出或控件重叠。

> 视觉验收未勾选：本地服务已在 `127.0.0.1:5199` 可达，但当前会话的 in-app Browser 后端列表为空，无法执行截图级验证。代码已使用固定最小高度、两列网格、两行截断和弹性收缩约束，并通过 TypeScript 与生产构建。
