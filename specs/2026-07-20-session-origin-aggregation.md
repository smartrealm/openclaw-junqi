# 会话来源聚合规格

## BUG-SESS-01 · 后台运行不再平铺为聊天会话

**当前**：侧栏只过滤子智能体，Cron 和梦境运行会进入日期分组。

**目标**：用户会话继续按日期展示；Cron、梦境、子智能体和明确的系统来源聚合到折叠的“后台活动”。

**验收**：

- [x] Cron、梦境和子智能体不出现在普通会话日期分组。
- [x] 后台活动按来源显示数量和运行状态。
- [x] 未知来源默认保留为普通会话，避免静默隐藏用户数据。

## BUG-SESS-02 · 保留官方会话来源字段

**当前**：映射丢弃 `origin`、`spawnedBy`、`status`、`hasActiveRun`。

**目标**：Chat store 保留分类所需的官方字段，不制造 OpenClaw 不支持的字段。

**验收**：

- [x] `sessions.list` 映射字段与 OpenClaw 官方响应同名。
- [x] 分类器优先使用结构化字段，key 仅用于官方结构识别和兼容兜底。

## BUG-SESS-03 · 单一分类入口

**当前**：Cron/Subagent 判断分散在多个模块。

**目标**：统一 helper 负责 key 解析、来源分类和后台隔离判断。

**验收**：

- [x] 侧栏、消息隔离和宠物后台状态复用同一 helper。
- [x] 官方 Cron/Subagent key、旧版兼容 key 和普通渠道 key 均有回归测试。
