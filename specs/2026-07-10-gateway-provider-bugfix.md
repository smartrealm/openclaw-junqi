# Gateway 与供应方配置修复规格

## BUG-01 · 原生启动优先

**Current**：本地端口不可达后直接进入 Docker 检查。

**Target**：先调用桌面托管原生启动并等待健康，失败后才进入 Docker。

**Acceptance**：

- [x] 系统服务未运行且 Docker 不可用时，原生 Gateway 仍可启动。
- [x] 原生启动错误写入环形日志并继续执行明确的兜底逻辑。

## BUG-02 · 损坏服务回退

**Current**：服务命令失败或端口超时直接结束重启。

**Target**：失败后释放重启状态，启动桌面托管 Gateway 并等待端口。

**Acceptance**：

- [x] 服务命令非零退出时执行托管启动。
- [x] 服务命令成功但端口未就绪时也执行托管启动。
- [x] 返回成功前端口必须真实可连接。

## BUG-03 · 日志可见

**Current**：状态对象的日志为空，Manager 快照丢弃日志，启动面板隐藏。

**Target**：Rust 最近日志被格式化并持续透传，恢复开始后立即显示。

**Acceptance**：

- [x] 状态快照包含最近 stdout/stderr。
- [x] 仅有恢复日志或正在恢复时面板可见。
- [x] 重启失败会进入错误界面而不是永久保持连接中。

## BUG-04 · 供应方模型同步

**Current**：供应方显式模型数组可以缺少已经启用的默认模型。

**Target**：保存时补齐相同供应方下缺失的模型声明，不覆盖已有运行时字段。

**Acceptance**：

- [x] 缺失模型按规范供应方 ID 补入。
- [x] 已有模型的名称、上下文、兼容性等字段保持不变。
- [x] 无关供应方模型不会被错误合并。
