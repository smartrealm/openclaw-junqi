# Windows OpenClaw 运行时迁移修复规格

## BUG-WRM-01

**当前**：迁移现有数据时 Node.js、Git 和 npm 缓存控件被禁用，后端拒绝路径变化。

**目标**：允许重新选择独立依赖位置，但工作区和内部运行时仍按迁移目标重映射。

**验收**：

- [x] 迁移请求接受互不重叠的 Node.js、Git、npm 缓存和 npm prefix 新路径。
- [x] 相对路径、根目录、父目录穿越和目录嵌套仍被拒绝。

## BUG-WRM-02

**当前**：新 npm prefix 被保存，但旧 OpenClaw 二进制继续运行。

**目标**：prefix 变化触发一次定向安装，新 prefix 中的 OpenClaw 验证成功后成为唯一保存选择。

**验收**：

- [x] 普通安装、原位置重装、迁移到配置 prefix 三种语义互不混用。
- [x] 迁移安装不因旧 OpenClaw 仍存在而跳过。
- [x] 成功后清除待迁移状态，失败时保留以便重试。

## BUG-WRM-03

**当前**：退出应用后无法可靠恢复未完成的 OpenClaw 包迁移。

**目标**：bootstrap 持久化待迁移状态，启动检测将其重新送入原生安装流程。

**验收**：

- [x] bootstrap 序列化往返保留待迁移状态。
- [x] 检测结果向前端暴露待迁移状态。
- [x] 迁移完成前不会自动恢复依赖旧二进制的 Gateway。

## BUG-WRM-04

**当前**：同目录重配仅在 Gateway HTTP 可达时停止旧运行时。

**目标**：任何原生运行时位置变化都先停止旧进程并确认端口释放。

**验收**：

- [x] Gateway 不可达但仍有运行时状态时也执行停止。
- [x] 停止失败时不保存新路径；原先可达的 Gateway 按旧配置恢复。

## BUG-WRM-05

**当前**：存储配置与 OpenClaw 安装可并发写 bootstrap，完成函数不校验安装请求是否仍为当前请求。

**目标**：两个操作共享互斥边界，且只允许匹配当前 prefix 的安装提交完成。

**验收**：

- [x] OpenClaw 安装持有 Gateway 全局操作锁。
- [x] 当前 prefix 选择变化时，旧安装不能清除待迁移状态。

## BUG-WRM-06

**当前**：便携目录存在兼容 Node.js、但缺少 npm 时仍跳过安装并可能回退系统 npm。

**目标**：显式便携 Node.js 与其 npm 始终作为同一个运行时验证和修复。

**验收**：

- [x] 缺少配套 npm 的便携 Node.js 状态报告 npm 不可用。
- [x] 安装命令会重新部署缺少 npm 的便携 Node.js。

## BUG-WRM-07

**当前**：终端启动器失败发生在待迁移状态清除之后。

**目标**：终端集成是迁移提交前的最后一个可失败步骤。

**验收**：

- [x] 待迁移期间可使用已验证的新二进制生成启动器。
- [x] 启动器同步失败时保留待迁移状态。
- [x] 启动器成功后才清除待迁移状态。

## BUG-WRM-08

**当前**：路径等价规则存在多个实现，迁移请求使用嵌套可选值表达。

**目标**：共享一个跨平台路径策略，并由具名事务对象提交重定位结果。

**验收**：

- [x] setup、storage 和 bootstrap 提交复用 `paths` 模块的路径身份 API。
- [x] 重定位提交顺序封装在一个具名方法中。
- [x] 共享路径策略具有等价、非等价、可选路径和目录重叠测试。
