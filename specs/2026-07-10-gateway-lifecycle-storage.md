# Gateway 单实例生命周期与存储迁移规格

## BUG-GL01/GL02 · 生命周期操作单飞

**当前**：restart、start、ensure、stop、Docker 各自管理并发。

**目标**：同一时刻最多一个生命周期写操作；后到的 ensure 等待并重新探测，不返回伪失败。

**验收**：

- [ ] 并发获取生命周期门闩时只有一个所有者。
- [ ] ensure 内部启动不会二次获取门闩而死锁。
- [ ] restart 兜底启动与外部 start 不能并发。

## BUG-GL03 · 应用单实例

**当前**：多个 JunQi 进程可各自管理 Gateway。

**目标**：第二个应用实例不创建第二套生命周期管理器，只唤醒现有主窗口。

**验收**：

- [ ] Tauri 注册单实例插件。
- [ ] 第二实例事件显示并聚焦主窗口。

## BUG-GL04/GL05 · 统一状态所有权

**当前**：状态只可靠描述托管 Child。

**目标**：状态同时描述生命周期和运行模式，所有命令在成功、失败、停止时更新它。

**验收**：

- [ ] start/restart/docker/stop 均产生明确模式转换。
- [ ] 子进程退出后不会继续报告 managed_child/running。

## BUG-ST01 · bootstrap 路径解析

**当前**：固定使用 `~/.openclaw`。

**目标**：固定系统配置目录中的 bootstrap 决定状态、配置和工作区；显式 `OPENCLAW_*` 环境变量保持最高优先级。

**验收**：

- [ ] 没有 bootstrap 时兼容 `~/.openclaw`。
- [ ] bootstrap 生效后所有 `paths` 调用读取新目录。
- [ ] bootstrap 本身不存放在可迁移状态目录中。

## BUG-ST02 · 首次启动迁移

**当前**：没有迁移入口。

**目标**：首次启动可继续使用默认目录、迁移到所选目录或创建新环境。

**验收**：

- [ ] 迁移持有生命周期独占锁并停止受管进程。
- [ ] 数据先复制到目标侧临时目录并完成文件数、总字节和主配置校验。
- [ ] 只有校验成功才切换 bootstrap。
- [ ] 失败时源目录和原 bootstrap 保持可用。
- [ ] UI 在运行时检测后、安装执行前完成路径决策，并展示迁移进度与错误。
- [ ] 存储步骤完成后恢复检测阶段选定的原有后续分支。
