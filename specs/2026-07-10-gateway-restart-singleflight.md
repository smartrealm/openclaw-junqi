# Gateway 重启单飞修复规格

## BUG-01 - 进程级重启所有权

现状：每个 IPC 调用都可能拉起 `openclaw gateway restart`。

目标：全局只有一个持有者执行；并发调用等待并读取它的最终状态。

验收：并发调用不能同时持有重启闸门；等待者不能创建新的重启子进程。

## BUG-02 - 前端重启所有权

现状：每个 retry 调用都独立调用 IPC。

目标：并发调用拿到同一个进行中的 Promise。

验收：两个并发 `run()` 只执行一次任务；任务结束后的新调用可以再次执行。

## BUG-03/04 - 正确的重启事件语义

现状：普通日志会设置 `running=false/retrying=true`；失败后繁忙状态可能不清除。

目标：诊断日志不改变生命周期；明确的开始/结束事件负责重启 UI 状态。

验收：`gateway-log` 只处理诊断内容；重启结束会清除繁忙状态并刷新真实状态；相邻重复日志不会追加两次。

## BUG-05 - 可用的恢复诊断窗口

现状：短日志窗口只能看到 6-8 行。

目标：桌面端至少保留 24 条近期记录，并提供明显更高的可视区域。

验收：离线页和启动页日志高度至少 224px；新输出自动保持可见。

## BUG-06 - 蓝色萌宠 UI 强调色

现状：萌宠控件和状态强调色复用龙虾橙。

目标：所有主题下的萌宠 UI 使用蓝色/青色强调色，龙虾本体仍为珊瑚色。

验收：深色主题 primary 不再是 `#ff836f`；浅色主题 primary 不再是 `#ef6f5e`；龙虾本体颜色测试保持不变。
