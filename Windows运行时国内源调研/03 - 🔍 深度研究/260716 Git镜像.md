# Git for Windows 国内镜像核验

**生成时间**：2026-07-16

## 核心结论

Git for Windows/MinGit 可用的可信国内镜像主要是 npmmirror 和华为云。阿里云、腾讯云、清华 TUNA 与中科大没有对应的 Git for Windows Releases 仓库。

## 可用镜像

- [npmmirror Git for Windows](https://registry.npmmirror.com/-/binary/git-for-windows/)：完整同步 Releases，并提供可机器读取的目录 JSON。
- [华为云 Git for Windows](https://mirrors.huaweicloud.com/git-for-windows/)：镜像 API 显示同步成功、一致率 100%，并覆盖 MinGit x64 与 ARM64。

以 `v2.55.0.windows.3` 为样本，两个镜像的 x64 MinGit 均为 38,791,206 字节，SHA-256 均为 `f48e2d2dc74a24454adc6d8fd0ac25bf9c2386f19cfb06202b9465aaad4f9f05`；ARM64 均为 37,446,324 字节，SHA-256 均为 `f7748965d5068e81ad93ca1923650db6742d6e22332b1ae7567a841c59f6bde5`。

## 信任边界

国内镜像只负责传输。JunQi 发布时审核 Git for Windows 版本和发布方摘要，并把固定版本及 SHA-256 写入代码；终端用户不请求 GitHub，也不能根据镜像目录自行信任一个未知新版本。
